# AGENTS.md

## 1. このプロジェクトについて

このリポジトリは、VRChatワールド「漂泊ノ海」へ手紙を投函するためのWebフォームです。

利用者が差出人、標題、本文を入力し、内容を確認したうえで投函します。

現在は静的なフロントエンドを中心に構築しており、今後Google Apps Script（GAS）およびGoogleスプレッドシートと連携します。

---

## 2. 技術構成

### フロントエンド

* HTML
* CSS
* Vanilla JavaScript
* GitHub Pagesで公開予定
* フレームワークやビルドツールは使用しない
* 外部ライブラリの追加は原則として行わない

### バックエンド

今後、以下を追加予定です。

* Google Apps Script
* Googleスプレッドシート
* Cloudflare Turnstile

## 3. 基本方針

既存のデザイン、文言、画面構成、入力動作を尊重してください。

依頼されていない箇所を、独自判断で変更しないでください。

特に以下を禁止します。

* 依頼されていないUI文言の変更
* 既存デザインの大幅な変更
* 既存機能の削除
* ファイル構成の大幅な変更
* ライブラリやフレームワークの無断追加
* 既存コード全体の不要な書き換え
* 「ついで」の仕様追加

必要な変更だけを、可能な限り小さな差分で実装してください。

## 4. 表示言語と命名

### 利用者向け表示

画面に表示する文章、エラーメッセージ、ボタン名は日本語とします。

既存の表示文言は、明示的な指示がない限り変更しないでください。

### ソースコード

変数名、関数名、クラス名、IDなどの識別子は英語を使用してください。

日本語の文字化けが発生しないようにしてください。

既存ファイルの改行コードや文字コードを、不必要に変更しないでください。

## 7. HTML実装ルール

既存のHTML構造を尊重してください。

タグの開始と終了の対応を確認し、閉じタグの不足や過剰を発生させないでください。

以下を意識してください。

* `label` と入力要素の関連付けを維持する
* `id` の重複を作らない
* `aria-*` 属性を不必要に削除しない
* ボタンには適切な `type` を指定する
* 装飾目的の画像には適切な空の `alt` を設定する
* キーボード操作を妨げない

---

## 8. CSS実装ルール

既存の配色、余白、フォント、角丸、半透明表現などの世界観を維持してください。

主なデザイン方針は以下です。

* 深い青を基調とする
* 白や淡い青を文字色に使用する
* 半透明パネルを使用する
* 見出しには主に `Noto Serif JP` を使用する
* 本文には主に `Zen Kaku Gothic New` を使用する
* 派手すぎる動きや強い原色を追加しない

新しいスタイルを追加する場合は、既存クラスとの競合を避けてください。

既存の共通クラスを変更する場合は、他の要素への影響を確認してください。

レスポンシブ表示を維持し、PCだけでなくスマートフォン表示も確認してください。

---

## 9. JavaScript実装ルール

Vanilla JavaScriptで実装してください。

依頼がない限り、外部JavaScriptライブラリは追加しないでください。

以下を守ってください。

* グローバル変数を不必要に増やさない
* 同じ処理を複数箇所へ重複させない
* 入力チェックを関数として整理する
* DOM要素が存在することを確認してから操作する
* イベントリスナーの重複登録を避ける
* 通信中の多重操作を防止する
* エラー時に利用者の入力内容を失わせない
* コンソールへ機密情報や入力本文を出力しない

既存機能を整理するためのリファクタリングは、必要最小限にしてください。

---

## 10. 入力チェック

ブラウザ側の入力チェックは、利用者への即時フィードバックを目的とします。

入力チェックには以下を含みます。

* 必須入力
* 最大文字数
* 同意チェック
* NGワード
* 必要に応じた入力形式チェック

ブラウザ側のチェックは、セキュリティ上の最終防御として扱わないでください。

将来GASと連携する場合は、GAS側でも必ず同じ条件を検証してください。

---

## 11. NGワード機能

NGワードは外部ファイルで管理します。

予定しているファイルは以下です。

`content/ng-words.json`

基本形式は以下です。

```json
{
  "words": []
}
```

`words` の内容は、以下のすべての入力欄へ共通で適用します。

* 差出人
* 標題
* 本文

NGワード一覧をHTMLやJavaScriptへ直接ハードコードしないでください。

NGワード判定時は、入力値とNGワードの両方に同じ正規化処理を適用してください。

想定する正規化は以下です。

* UnicodeのNFKC正規化
* 英字の小文字化
* 半角・全角スペースの除去
* タブおよび改行の除去

一致した具体的なNGワードは利用者へ表示しないでください。

エラー表示は以下とします。

`使用できない単語が含まれています。内容をご確認ください。`

ブラウザ側のNGワード判定は容易に回避できるため、将来GAS側にも同等の判定を実装してください。

---

## 12. 外部ファイルの読み込み

JSONやHTMLなどの外部ファイルを `fetch()` で読み込む場合は、以下を考慮してください。

* 読み込み成功前に関連機能を有効化しない
* 読み込み失敗時の処理を実装する
* 必要に応じてキャッシュ制御を行う
* JSON形式が不正な場合もエラーとして扱う
* エラー発生時は安全側へ倒す

たとえばNGワード一覧の読み込みに失敗した場合、投稿を許可しない設計とします。

---

## 13. GAS連携時のルール

GASとの連携を実装する場合、以下を守ってください。

* ブラウザから送られた値を信用しない
* GAS側でも必須チェックを行う
* GAS側でも文字数を確認する
* GAS側でもNGワードを判定する
* GAS側で同意状態を確認する
* 不正な形式のデータを拒否する
* 保存成功後だけ成功レスポンスを返す
* エラー内容を利用者へ過剰に公開しない
* スプレッドシートIDや秘密情報をフロント側へ書かない
* シークレットキーをHTMLやJavaScriptへ書かない

GAS側のシークレット値はScript Properties等で管理してください。

---

## 14. Cloudflare Turnstile

Cloudflare Turnstileを導入する場合は、ブラウザ側へウィジェットを表示するだけでは不十分です。

以下を必須とします。

1. ブラウザでTurnstileトークンを取得する
2. 投稿内容と一緒にGASへ送信する
3. GASからCloudflareへトークン検証を行う
4. 検証成功時のみスプレッドシートへ保存する

Turnstileのシークレットキーをフロント側へ記述しないでください。

Turnstileに失敗した場合やトークンがない場合は、投稿を保存しないでください。

---

## 15. 投稿処理

本番の投稿処理は以下の順番とします。

1. ブラウザ側で入力を検証
2. 投函確認画面を表示
3. 利用者が最終確定
4. 投函ボタンを無効化
5. GASへ投稿内容を送信
6. GAS側で再検証
7. 必要に応じてTurnstileを検証
8. スプレッドシートへ保存
9. 保存成功レスポンスを受信
10. 投函演出を開始
11. 投函完了画面を表示

保存に失敗した状態で、投函完了画面を表示しないでください。

通信失敗時は入力内容を維持し、再試行できるようにしてください。

---

## 16. 二重投稿防止

以下の対策を行ってください。

* 送信開始後は送信ボタンを無効化する
* 通信中に同じ送信処理を複数回実行しない
* 保存成功後に同じデータを再送信しない
* 必要に応じて投稿IDや一意な識別子を利用する

単にボタンを無効化するだけで、サーバー側の二重投稿対策が不要になるわけではありません。

---

## 17. エラー表示

エラーは、利用者が次に何をすればよいか分かる日本語で表示してください。

技術的な例外内容、スタックトレース、内部URL、シークレット情報は利用者へ表示しないでください。

入力エラーは原則として該当欄の近くへ表示してください。

複数の検証エラーを管理する場合、あるエラーを表示したことで別のエラー状態が消えないようにしてください。

---

## 18. セキュリティ

以下を前提としてください。

* HTMLとJavaScriptは利用者から閲覧・改変可能
* GASのURLへ直接リクエストを送信される可能性がある
* フロント側の入力チェックは回避可能
* 公開JSONファイルは誰でも閲覧可能
* hidden入力やdisabled属性はセキュリティ対策にならない

セキュリティ上重要な判定は、必ずGAS側でも実施してください。

## 20. テスト

変更後は、最低限以下を確認してください。

### 共通

* HTMLタグの対応に問題がない
* JavaScriptエラーが発生していない
* PC表示が崩れていない
* スマートフォン表示が崩れていない
* ページ内リンクが動作する
* キーボード操作が可能
* 既存アニメーションが壊れていない

### 入力フォーム

* 必須項目が未入力の場合にエラーになる
* 最大文字数が正しく設定されている
* 文字数表示が正しく更新される
* 同意チェックが機能する
* 下書き保存と復元が機能する
* 下書き削除が機能する
* 確認モーダルが機能する
* 入力内容が確認画面へ正しく表示される

### NGワード

* 各入力欄でNGワードが検出される
* NGワードを削除するとエラーが解除される
* 空白を挟んだ場合も検出される
* 全角と半角の違いが正規化される
* NGワードがある場合は確認画面へ進めない
* NGワードJSONの読み込み失敗時は投稿できない

### GAS連携後

* 正常投稿が正しい列へ保存される
* 保存失敗時は完了画面を表示しない
* 通信失敗時も入力内容が残る
* 二重送信されない
* 不正な直接リクエストが拒否される
* Turnstile失敗時は保存されない
